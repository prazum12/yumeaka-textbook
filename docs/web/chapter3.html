第3章

CSSでデザイン 〜ページを着せかえる〜

📅 第3回の講習で使います
💻 使っているPC:

🎯 この章でできるようになること

  • CSSファイルを作って、HTMLとつなげられる
  • セレクタ・プロパティ・値を使って、色・文字・背景を自由に変えられる
  • Flexbox（フレックスボックス）でナビバーやカードを横に並べられる

第2章で作った「好きなもの紹介ページ」、中身はそろってきたけど、見た目は白い背景に黒い文字だけだったよね。
今日はCSSの出番。
ページに色をつけて、文字を飾って、カードを横に並べて、 「それっぽいWebサイト」に大変身させよう。

アオイ色を変えたら、急にデザインっぽくなる？

レン色だけじゃなく、余白と並び方が主役。まず1か所変えて、ブラウザで比べよう。

3-1. CSSってなに？
〜HTMLが骨、CSSが服〜

第1章で「HTMLは骨、CSSは服や髪型」とたとえたのを覚えているかな。
HTMLはページに「何をのせるか」を決めるだけで、「どう見せるか」はすべてCSS（シーエスエス）の仕事です。
同じHTMLでも、CSSを変えるだけで見た目はガラッと変わります。

CSS … Cascading Style Sheets（カスケーディング・スタイル・シート）の略。
ページの「見た目のルール」を書くための言語。
HTMLとは文法がぜんぜん違うので、別の言語として覚えよう。

3-2. style.css を作ってHTMLとつなげよう

CSSは、HTMLとは別のファイルに書くのが基本です。
style.css というファイルを作って、HTMLから「このCSSを使うよ」と宣言（リンク）します。
1つのCSSファイルを何ページからでも使い回せる、とても便利な仕組みです。

1style.css を作る

VSCodeで yumeaka-web フォルダを開き、エクスプローラーの「新しいファイル」アイコンをクリック。
style.css と入力してEnter。
index.html と同じ場所（imgフォルダの中ではない）に作ろう。

2HTMLに link タグを書く

index.html を開いて、<head> の中に次の1行を追加します。
これが「このページは style.css を使います」という宣言です。

index.html（headの中に追加） 
<link rel="stylesheet" href="style.css">
3つながったかテストする

style.css に次のコードを書いて、両方のファイルを保存（Ctrl + S⌘ + S）。
ブラウザで見出しがピンクになったら接続成功！

style.css 
h1 {
  color: deeppink;
}
linkタグ … HTMLに別のファイルを結びつけるタグ。
rel="stylesheet" は「結びつける相手はスタイルシート（CSS）です」という意味。
<img> と同じく終了タグのない仲間。

🚨 つまずきポイント: 保存したのにピンクにならない

まず index.html と style.css の両方を保存したか確認。
次に href="style.css" のつづりがファイル名と1文字も違わないかチェックしよう。

3-3. CSSの文法 〜セレクタ・プロパティ・値〜

さっき書いた h1 { color: deeppink; } を分解してみよう。
CSSはどんなに長くなっても、ぜんぶ「どこを・何を・どう変える」の組み合わせでできています。

セレクタ … 「選ぶもの」という意味。
h1 と書けばページ内のすべての h1 が対象になる。
{ } の中には指示を何行でも書けるよ。

名前をつけてねらいうち 〜classセレクタ〜

p { color: red; } と書くと、ページ中のすべての段落が赤くなってしまいます。
「この段落だけ」を選びたいときは、HTML側で class 属性で名前をつけて、CSS側で .名前 で選びます。

HTML側: classで名前をつける 
<p class="notice">ここだけ目立たせたい！</p>
<p>こっちはふつうの段落。</p>
CSS側: .名前 で選ぶ 
.notice {
  color: red;
  font-weight: bold;
}
class（クラス） … タグにつける「あだ名」。
CSSでは頭に .（ドット）をつけて選ぶ。
名前は半角英数字で、自分でわかりやすいものを自由につけてOK。
同じclassを何個のタグにつけてもいい。

3-4. 色・文字・背景を変えよう

まずはよく使うプロパティをまとめて覚えよう。
この5つだけでも、ページの印象は大きく変えられます。

プロパティ意味値の例
color文字の色deeppink、#0ea5e9
background-color背景の色lightyellow、#f0f9ff
font-size文字の大きさ20px、32px
font-weight文字の太さbold（太字）
text-align文字の寄せ方center（中央寄せ）
style.css（ページ全体と見出しを飾る例） 
body {
  background-color: #f0f9ff;   /* ページ全体のうすい背景色 */
}

h1 {
  color: #0369a1;
  font-size: 40px;
  text-align: center;
}

body をセレクタにするとページ全体に効きます。
ちなみにCSSのコメント（メモ）は /* 〜 */ で書きます。
HTMLの <!-- --> とは形が違うので注意。

カラーコード … #0ea5e9 のように #+6桁で色を表す書き方。
red や skyblue のような色名より細かく指定できる。
Googleで「カラーピッカー」と検索すると、好きな色のコードをその場で調べられるよ。
px（ピクセル） … 画面の点の数で大きさを表す単位。
font-size: 20px; なら高さ20個分の点で文字を描く。
数字を変えて、ちょうどいい大きさを探そう。

3-5. ボックスモデル 〜すべての要素は「箱」〜

CSSの世界では、h1 も p も img も、すべての要素が四角い箱でできています。
そして箱には「余白」と「枠線」の層があります。
この仕組みをボックスモデルと呼びます。
レイアウトが思い通りにならないときは、だいたいこの箱のどこかが原因です。

style.css（見出しを「箱」として飾る例） 
h2 {
  background-color: #0ea5e9;   /* 箱の背景色 */
  color: white;
  padding: 12px;               /* 枠と文字のすき間 */
  border: 3px solid #0369a1;   /* 太さ 線の種類 色 */
  border-radius: 10px;         /* 角を丸くする */
  margin: 30px 0;              /* 上下30px・左右0の外余白 */
}

保存してブラウザを見ると、h2 がバナーのような見た目になったはず。
padding や margin の数字をいろいろ変えて、箱がどう動くか実験してみよう。
値を2つ書くと「上下・左右」の順で指定できます。

3-6. Flexboxで横に並べよう

HTMLの要素は、ふつう上から下へ縦に積まれていきます。
でもナビゲーションメニューやカードは横に並べたいよね。
そこで使うのが Flexbox（フレックスボックス）。
並べたいものたちを「親の箱」に入れて、親に display: flex; と書くだけです。

div（ディブ） … 意味を持たない「ただの箱」のタグ。
いくつかの要素をグループにまとめて、CSSで飾ったり並べたりするために使う。
Flexboxの「親の箱」としても大活躍。

作ってみよう①: ナビバー（メニュー）

HTML（bodyのいちばん上あたりに） 
<div class="navbar">
  <a href="index.html">ホーム</a>
  <a href="favorite.html">好きなもの</a>
  <a href="#">プロフィール</a>
</div>
style.css 
.navbar {
  display: flex;               /* 子どもを横に並べる */
  gap: 24px;                   /* 子ども同士のすき間 */
  background-color: #0369a1;
  padding: 14px 20px;
}

.navbar a {
  color: white;
  font-weight: bold;
}

.navbar a は「navbar クラスの中にある a タグ」という意味。
セレクタを半角スペースでつなぐと「〜の中の〜」と絞りこめます。

作ってみよう②: カードを横に並べる

HTML 
<div class="cards">
  <div class="card">
    <h3>ゲーム</h3>
    <p>マインクラフトにハマっています。</p>
  </div>
  <div class="card">
    <h3>食べもの</h3>
    <p>海軍カレーが大好きです。</p>
  </div>
  <div class="card">
    <h3>音楽</h3>
    <p>最近はロックをよく聴きます。</p>
  </div>
</div>
style.css 
.cards {
  display: flex;
  gap: 16px;
}

.card {
  flex: 1;                     /* 3枚の幅を等分する */
  background-color: white;
  border: 2px solid #0ea5e9;
  border-radius: 12px;
  padding: 16px;
}

Flexboxでよく使う仲間もまとめておくね。
ぜんぶ親の箱に書くプロパティだよ（flex: 1 だけは子に書く）。

プロパティできること
display: flex;子どもたちを横に並べる（すべての出発点）
gap: 16px;子ども同士のすき間をあける
justify-content: center;子どもたちを横方向の中央に寄せる
align-items: center;子どもたちを縦方向の中央にそろえる
flex: 1;（子に書く）子どもたちの幅を等分にする

3-8. 🚨 つまずきポイント集

CSSを書いたのに、ページが何ひとつ変わらない

原因: HTMLとCSSがつながっていない。
link タグの書き忘れ・書き間違いが最有力。

直し方: ①<head> の中に link タグがあるか ②href の値がファイル名と完全一致か（style.css と styles.css は別物！）
③rel="stylesheet" を書いたか、の順に確認。

途中までは効くのに、ある行から下のCSSが全部無視される

原因: その行より上に } の閉じ忘れか ; の書き忘れがある。
1か所こわれると、そこから下が道連れになるのがCSS。

直し方: 効かなくなった場所のすぐ上のブロックの { } と ; を確認。
VSCodeで { をクリックすると、相方の } を教えてくれるよ。

1か所だけ効かない（ほかは効いているのに）

原因: セレクタ名やプロパティ名のスペルミス。
classセレクタの .（ドット）忘れも多い。

直し方: HTML側の class="cards" とCSS側の .cards が1文字も違わないか見比べる。
colour（❌）や background-colar（❌）にも注意。

横に並ばない！
縦のまま！

原因: display: flex; を並べたい要素自身（子）に書いている。

直し方: 書く場所は親の箱（図3-4）。
並べたいものたちが1つの div にちゃんと包まれているかも確認しよう。

色コードが効かない・変な色になる

原因: # のつけ忘れ、6桁に足りない、0（ゼロ）と o（オー）の見間違いなど。

直し方: カラーピッカーからコピペし直すのが確実。
VSCodeはCSSの色コードの左に小さな色見本を出してくれるので、思った色になっているか見てみよう。

3-9. 🤖 AIを味方につけよう

この章でのAIの使い所

✅ 良い使い方 — まず15分自分で考えてから

  • 「CSSが効かない」の原因さがし: HTMLとCSSの両方を貼って確認手順を聞く（linkミス・スペルミスの発見が得意）
  • 配色の相談: 好きな色を伝えて、読みやすい組み合わせを色コード付きで提案してもらう
  • プロパティの逆引き: 「文字に影をつけたい」など、やりたいことからプロパティ名を教えてもらう

そのままコピペで使えるプロンプト（AIへの指示文）の例:

CSSを勉強中の中学生です。
style.cssに書いたのに見た目が変わりません。
HTMLとCSSを貼ります:（両方のコードを貼る）。
原因として考えられるものを、確認する順番に教えてください。
答えのコードだけでなく「なぜそうなるか」も知りたいです。
水色が好きなので、水色をメインにしたWebページの配色を考えています。
背景色・文字色・アクセント色の組み合わせを、カラーコード付きで3パターン提案してください。
文字が読みやすいことを大事にしてください。

🚫 やめておこう

  • 「かっこいいデザインのCSSを全部書いて」と丸投げする（それはAIの作品。
    自分で色や数字をいじるから楽しいし力がつく）
  • AIがくれたCSSを動かして確かめずに貼りつける（必ずブラウザで見た目を確認！）
