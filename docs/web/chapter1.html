第1章

環境構築とHTML入門

📅 第1回の講習で使います
💻 使っているPC:

🎯 この章でできるようになること

  • Webサイトが表示される仕組みを説明できる
  • VSCode（プログラミング用のエディタ）を自分のPCにインストールできる
  • HTMLファイルを作って、ブラウザに「自分の言葉」を表示できる

アオイ英語みたいなコード、全部覚えないとだめ？

レン覚えるより、保存→表示→直すの小さなループを回そう。今日は自分の言葉がブラウザに出ればクリア。

1-1. Webサイトってどうやって表示されるの？

みんなが毎日見ているYouTubeもゲームの攻略サイトも、すべて「Webサイト」です。
Webサイトは、大きく分けて 3つの言語 でできています。

人間にたとえると、HTMLは骨、CSSは服や髪型、JavaScriptは動きです。
まずは骨組みのHTMLから始めます。

ブラウザ … Webサイトを見るためのアプリ。
Chrome（クローム）、Safari（サファリ）、Edge（エッジ）など。
この講習ではChromeを使います。

1-2. 道具をそろえよう（VSCodeのインストール）

プログラミングのコードは、エディタという専用のメモ帳アプリで書きます。
世界中のプロが使っている無料エディタ「Visual Studio Code（VSCode）」をインストールしましょう。

1公式サイトからダウンロード

ブラウザで code.visualstudio.com を開き、まん中の大きなダウンロードボタンを押します。
自分のPC（Windows / Mac）に合ったものが自動で選ばれます。

2インストールする

ダウンロードしたファイルをダブルクリック →「同意する」を選ぶ → あとは「次へ」を押していけばOK。

ダウンロードしたzipを開き、出てきた「Visual Studio Code」を「アプリケーション」フォルダにドラッグ。

3日本語にする

VSCodeを起動したら、左端の四角が4つ並んだアイコン（拡張機能）をクリック。
検索欄に japanese と入力し、「Japanese Language Pack」の Install を押します。
右下に出る「Change Language and Restart」を押すと日本語になります。

4Live Server を入れる

同じ手順で live server と検索し、「Live Server」（作者: Ritwick Dey）をインストールします。
書いたコードをすぐブラウザで確認できる便利な道具です。

🚨 つまずきポイント: 「Install」ボタンが見つからない

検索結果に似た名前がたくさん出ます。
ダウンロード数（数字）がいちばん多いものを選ぼう。

1-3. 作業フォルダを作ろう

コードのファイルは1つのフォルダにまとめて管理します。

1フォルダを作る

デスクトップに yumeaka-web という名前のフォルダを作ります（右クリック → 新規フォルダ）。
フォルダ名やファイル名は半角英数字にするのがプログラミングの約束です。

2VSCodeでフォルダを開く

VSCodeの左上の「ファイル」メニュー画面上部の「ファイル」メニューから「フォルダーを開く」→ さっき作った yumeaka-web を選択。
「作成者を信頼します」と聞かれたら「はい」でOK。

1-4. はじめてのHTMLファイル

1ファイルを作る

左側のエクスプローラーで「新しいファイル」アイコンをクリックし、index.html と入力してEnter。
「.html」まで正確に入力しよう。

2コードを書く

次のコードを自分の手で打ち込んでみよう（コピーもできるけど、最初は打つのが上達の近道！）。

index.html 
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8">
    <title>はじめてのページ</title>
  </head>
  <body>
    <h1>こんにちは、世界！</h1>
    <p>これは私がはじめて作ったWebページです。</p>
  </body>
</html>

書けたら Ctrl + S⌘ + S で保存。
タブの「●」が消えたら保存成功です。

3ブラウザで表示する

コードの上で右クリック →「Open with Live Server」を選ぶと、ブラウザが自動で開いて自分のページが表示されます！

コードの意味をひとつずつ

コード意味
<!DOCTYPE html>「これはHTMLの文書です」という宣言。
おまじないとして毎回最初に書く
<html>〜</html>HTML全体を包む入れ物
<head>〜</head>ページの設定を書く場所（画面には表示されない）
<title>ブラウザのタブに表示されるタイトル
<body>〜</body>画面に表示される内容を書く場所
<h1>いちばん大きな見出し（heading 1）
<p>文章の段落（paragraph）
タグ … <h1> のように < > で囲まれた命令のこと。
<h1>（開始タグ）と </h1>（終了タグ）で内容をはさんで使う。
/ を忘れやすいので注意！

1-5. 💪 やってみよう

課題1: 自分の言葉に変えよう

<h1> の中身を自分の好きな言葉（座右の銘、好きなゲームのセリフなど）に変えて、保存 → ブラウザで確認しよう。
Live Serverなら保存した瞬間に画面が変わるよ。

課題2: 段落を増やそう

<p> タグを使って、「今ハマっていること」を3つ、3つの段落で追加してみよう。

ヒントを見る

<p>サッカーにハマっています。</p> のような行を、</body> より上に3行書けばOK。

課題3（チャレンジ）: 見出しのサイズを調べよう

見出しタグは <h1> から <h6> まであります。
全部書いて表示して、大きさがどう変わるか確かめよう。

✅ お手本を見る — まずは自分でやってみて、できたら答え合わせに使おう。
課題1〜3をやり終えるとこんな表示になるよ。

1-6. 🚨 つまずきポイント集

画面に <h1> などのタグがそのまま表示される

原因: ファイル名が .html で終わっていない（.txt になっている等）。

直し方: エクスプローラーでファイル名を右クリック → 名前の変更で index.html に。

保存したのにブラウザの表示が変わらない

原因1: 保存できていない。
タブに「●」が付いていたら未保存。
Ctrl + S⌘ + S を押そう。

原因2: Live Serverを使わずファイルを直接開いている。
その場合はブラウザの更新ボタン（F5）を押そう。

日本語が文字化けする（□や記号だらけになる）

原因: <meta charset="UTF-8"> の書き忘れ・打ち間違い。

直し方: <head> の中にこの1行があるか、スペルが正しいか確認しよう。

途中から表示がおかしい・全部太字になった

原因: 終了タグの / 忘れが一番多い。

直し方: 開始タグと終了タグがペアになっているか、上から順に確認。
VSCodeはタグをクリックするとペアの相手を薄く光らせてくれるよ。

💡 15分たっても直らないときは、次の「AIを味方につけよう」の方法で相談してみよう。

1-7. 🤖 AIを味方につけよう

AIサービスには、それぞれ利用できる年齢の条件があります。2026年7月時点では、ChatGPTは13歳以上で、18歳未満は保護者または法定代理人の許可が必要です。Claude.aiは18歳以上です。利用前に保護者と一緒に、ChatGPTの利用規約とClaude.aiの利用規約の最新版を確認してください。
ただし使い方しだいで「上達が速くなる」ことも「まったく力がつかない」こともあるので、この教科書ではおすすめの使い方を毎章で紹介します。

この章でのAIの使い所

✅ 良い使い方 — 自分で15分考えてわからなかったら聞く

  • エラーや文字化けの相談: 画面の様子とコードを伝えて原因を聞く
  • 用語の解説: わからない言葉を自分のレベルに合わせて説明してもらう
  • 練習問題を出してもらう: 覚えたタグの復習に

そのままコピペで使えるプロンプト（AIへの指示文）の例:

HTMLを勉強中の中学生です。
index.htmlをブラウザで開いたら日本語が「□□□」のように文字化けしました。
コードはこれです: （自分のコードを貼る）。
原因と直し方を、専門用語を使いすぎずに教えてください。
「タグ」と「要素」の違いがわかりません。
HTMLを始めたばかりの中学生にわかるように、たとえ話で説明してください。
