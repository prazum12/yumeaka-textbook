JavaScript入門 〜コンピュータに命令しよう〜

📅 第5回の講習で使います
💻 使っているPC:

🎯 この章でできるようになること

  • プログラムが「上から順に1行ずつ実行される」ことを説明できる
  • 開発者ツールのコンソールを開いて、console.log の結果を確かめられる
  • 変数とif文を使って「年齢でメッセージが変わる」プログラムを書ける

第4章では、HTMLとCSSを使いこなして自己紹介サイトを完成させたね。
骨組み（HTML）と見た目（CSS）ときたら、残るは3つ目の言語 —— ページに動きをつける JavaScript（ジャバスクリプト）です。
この章ではその第一歩として、「コンピュータに命令する」というプログラミングの基本を体験しよう。

アオイJavaScriptは今までのHTMLと何が違うの？

レンHTMLは部品、JavaScriptは命令。実行前に結果を予想してから動かすと、考え方が身につくよ。

5-1. プログラミングとは？
〜コンピュータへの指示書〜

プログラムとは、コンピュータにやってほしいことを順番に書いた指示書です。
料理のレシピとそっくりで、コンピュータは書かれた命令を上から順に、1行ずつ実行していきます。
これを順次実行（じゅんじじっこう）と呼びます。

「順番どおりに実行される」——当たり前に聞こえるけど、これがプログラミングのいちばん大事なルール。
あとで出てくるif文は、この流れを「条件によって枝分かれ」させる仕組みだよ。

JavaScript … ブラウザの中で動くプログラミング言語。
ページに動きをつけたり、計算や判断をさせたりできる。
名前が似ている「Java」はまったく別の言語なので注意！

5-2. はじめてのJavaScript

JavaScriptのコードは .js で終わるファイルに書き、HTMLから <script> タグで読み込みます。
さっそく練習用のファイルを2つ作ってみよう。

1js.html を作る

yumeaka-web フォルダに新しいファイル js.html を作って、次のコードを書こう。

js.html 
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8">
    <title>JavaScriptの練習</title>
  </head>
  <body>
    <h1>JavaScriptの練習</h1>
    <p>結果はコンソールに表示されます。</p>
    <script src="script.js"></script>
  </body>
</html>

最後の行の <script src="script.js"></script> が、JavaScriptファイルを読み込む命令。
画像の src と同じ「相対パス」だね。
</body> の直前に書くのがお約束です（ページの内容を先に読み込んでから動かすため）。

2script.js を作る

同じフォルダに script.js を作って、はじめてのプログラムを書こう。

script.js 
console.log("こんにちは！");
console.log("JavaScriptの世界へようこそ");

console.log(〜) は「カッコの中身をコンソールに表示して」という命令。
保存して js.html を右クリック →「Open with Live Server」で開いてみよう。
……あれ、画面には何も増えていない？
それで正解！
結果はコンソールという秘密の画面に出ています。
次で開き方を覚えよう。

セミコロン（ ; ） … 命令の終わりにつける記号。
日本語の「。」
にあたる。
なくても動くことが多いけど、「1つの命令を書いたら ; 」のクセをつけておくとエラーに強くなるよ。

5-3. コンソールを開こう 〜開発者ツール〜

ブラウザには、プログラマー向けの隠し機能開発者ツール（デベロッパーツール）が入っています。
その中のコンソールは、console.log の結果やエラーメッセージが表示される場所。
JavaScriptを書くときの相棒になるので、開き方をしっかり覚えよう。

1開発者ツールを開く

ページのどこかを右クリックして、メニューのいちばん下にある「検証」をクリック。
キーボード派は F12 キー（または Ctrl + Shift + I）でもOK。 キーボード派は ⌘ + option + I でもOK。

2「コンソール」タブに切りかえる

開発者ツールの上部に並ぶタブから「コンソール」（Console）をクリック。
こんにちは！ と JavaScriptの世界へようこそ が表示されていたら大成功！
君のプログラムがちゃんと実行された証拠です。

🚨 つまずきポイント: 開発者ツールが英語で表示される

タブが「Console」「Elements」のように英語でも中身は同じ。
「Console」を押せばOK。
（日本語にしたい人は、開発者ツール右上の歯車 → Language → Japanese）

5-4. 変数 〜データに名前をつける箱〜

プログラムでは、数字や文字などのデータに名前をつけて保存しておけます。
この仕組みが変数（へんすう）。
「名前つきの箱」をイメージすると分かりやすいよ。

変数の作り方は2種類。
あとで中身を入れかえられる let と、入れかえ禁止の const（コンスト）です。

script.js（書きかえて試そう） 
let score = 0;          // let は中身を入れかえOK
console.log(score);     // → 0
score = 100;            // 2回目からは let を書かない
console.log(score);     // → 100

const school = "夢アカデミー";  // const は入れかえ禁止
console.log(school);    // → 夢アカデミー
let / const … 変数を作る（宣言する）キーワード。
迷ったらまず const、あとで入れかえたくなったら let に変えるのがプロの流儀。
同じ名前の箱は2回作れないよ。
コメント … // から行末までは実行されないメモ。
HTMLの <!-- --> のJavaScript版だね。

5-5. 計算と文字列

JavaScriptは超高性能な電卓でもあります。
算数の記号とちょっと違うところに注意。

記号意味例結果
+たし算10 + 313
-ひき算10 - 37
*かけ算（×ではない！）10 * 330
/わり算（÷ではない！）10 / 42.5
%わり算の余り10 % 31
計算の例 
const price = 120;   // 1本の値段
const count = 3;     // 本数
console.log(price * count);         // → 360
console.log(price * count + 500);   // → 860（計算の順番も算数と同じ）

"こんにちは！" のように "（ダブルクォート）で囲んだデータは文字列（もじれつ）と呼びます。
文字列どうしは + でつなげることができて、変数と組み合わせると便利！

文字列をつなげる 
const name = "ゆめこ";
const age = 14;
console.log("わたしは" + name + "、" + age + "歳です");
// → わたしはゆめこ、14歳です
文字列 … "〜" で囲んだ文字のデータ。
囲み忘れると、JavaScriptはそれを変数の名前だと思ってエラーになる。
14（数値）と "14"（文字列）は別モノなのも大事なポイント。

5-6. if文 〜条件で枝分かれさせる〜

ここからが今日のメインイベント。
if文（イフぶん）を使うと、 「もし〜なら A、そうでなければ B」という判断をプログラムにさせられます。
順次実行の一本道が、条件によって枝分かれするイメージだよ。

if文の基本形 
const age = 14;

if (age >= 13) {
  console.log("13歳以上だね！");
} else {
  console.log("12歳以下だね！");
}

if (条件) { 〜 } の { }（波カッコ）の中が「条件がYESのときだけ」実行され、 else { 〜 } の中は「NOのとき」だけ実行されます。
age の値をいろいろ変えて、結果が変わるのを確かめてみよう。

条件に使える比較の記号はこちら。

書き方意味例（age が 14 のとき）
>=〜以上age >= 13 → YES
<=〜以下age <= 12 → NO
>〜より大きいage > 14 → NO
<〜より小さいage < 15 → YES
===等しい（=を3つ！）age === 14 → YES
!==等しくないage !== 14 → NO
= と === のちがい … = は「箱に入れる」、=== は「等しいか比べる」。
役割がまったく違う！
if の条件に = を1つだけ書くのは定番ミスなので要注意。

分かれ道を3つ以上にしたいときは、else if を間にはさみます。
上から順にチェックされて、最初にYESになったところだけが実行されるよ。

else if で3分岐 
const score = 85;

if (score >= 80) {
  console.log("すごい！");
} else if (score >= 50) {
  console.log("合格！");
} else {
  console.log("もうひといき！");
}

5-7. 💪 やってみよう

課題1: 自己紹介プログラム

変数 name（名前）と age（年齢）を作って、 コンソールに「わたしは〇〇、△△歳です」と表示するプログラムを script.js に書こう。

ヒントを見る

5-5の「文字列をつなげる」の例がほぼそのまま使えるよ。
名前は文字列だから "〜" で囲む、年齢は数値だからそのまま書く。

課題2: 年齢でメッセージを変えよう（この章のメイン課題！）

変数 age の値によって、表示されるメッセージが変わるプログラムを作ろう。

  • 12以下 →「小学生かな？」
  • 13〜15 →「中学生だね！」
  • 16以上 →「高校生以上だね！」

できたら age を 10、14、17 に変えて、3パターンとも正しく出るかテストしよう。
メッセージの言葉は自由にアレンジしてOK！

ヒント1: 形を見る

5-6の「else if で3分岐」の例と同じ形。
score を age に置きかえて考えよう。

ヒント2: 条件の順番

大きい数字から順にチェックするのがコツ。
if (age >= 16) → else if (age >= 13) → else の順なら、14歳は最初の条件でNO、次でYESになる。

課題3（チャレンジ）: 「かつ」の条件

「13歳以上、かつ18歳以下」のときだけ「夢アカデミーの対象年齢だね！」
と表示するには？
条件を2つ組み合わせる書き方 &&（アンド）を調べて挑戦してみよう。

ヒントを見る

if (age >= 13 && age <= 18) { 〜 } のように、&& で条件をつなぐと「両方YESのときだけYES」になる。
ちなみに ||（オア）は「どちらかがYESならYES」だよ。

開いたら開発者ツールのコンソール（5-3参照）を見てね。課題1〜3の結果が表示されているよ。

5-8. 🚨 つまずきポイント集

コンソールに何も表示されない

原因1: <script src="script.js"></script> の書き忘れ・ファイル名の打ち間違い。

原因2: script.js を保存していない（タブの「●」をチェック）。

直し方: scriptタグのつづりと src のファイル名を確認 → 両方のファイルを保存 → ブラウザを再読み込み。
コンソールに赤いエラーが出ていないかも見よう。

赤い文字で「Uncaught SyntaxError: Invalid or unexpected token」

原因: 全角文字がまぎれこんでいる（日本語入力のまま打った （） ” や全角スペースなど）。
日本語ユーザーの定番エラー！

直し方: エラーメッセージの右端に出る script.js:2 は「script.jsの2行目」という意味。
その行の記号を半角で打ち直そう。
記号を打つときは入力を半角英数（半角/全角キー英数キー）に切りかえるクセをつけると防げるよ。

赤い文字で「Uncaught ReferenceError: 〜 is not defined」

原因: スペルミス。
consol.log（eが足りない）や、変数名の打ち間違い（Age と age は別モノ！）。
文字列の "〜" の囲み忘れでも出る。

直し方: 「〜 is not defined（〜なんて知らないよ）」の「〜」の部分に、間違えたつづりがそのまま表示されている。
正しいつづりと見比べよう。

if文がいつも同じ結果になる

原因: 条件のところに = を1つだけ書いている（if (age = 13)）。
これは「比較」ではなく「代入」になってしまう。

直し方: 等しいかどうかは ===（3つ）。
「以上・以下」は >= <= を使おう。

計算のはずが「105」のように文字がくっついて出る

原因: "10" + 5 のように、数値のつもりが文字列になっている。
文字列の + は「つなげる」だったね。

直し方: 計算したい数字は " で囲まない。
const age = "14"; ではなく const age = 14; と書こう。

5-9. 🤖 AIを味方につけよう

この章でのAIの使い所

✅ 良い使い方 — まず15分自分で考えてから

  • エラーメッセージの翻訳: コンソールの赤い英語をコピーして「どういう意味？」
    と聞く（JavaScript学習の最強の使い方！）
  • 条件の書き方相談: 「〜のときだけ、はどう書く？」
    と考え方から聞く
  • コードの音読: 自分のコードを貼って「1行ずつ何をしているか説明して」と頼むと理解が深まる

そのままコピペで使えるプロンプト（AIへの指示文）の例:

JavaScriptを勉強中の中学生です。
コンソールに「Uncaught SyntaxError: Invalid or unexpected token script.js:3」と赤いエラーが出ました。
コードはこれです:（script.jsの中身を貼る）。
エラーの意味と、どこを直せばいいかを、答えのコードをいきなり書かずにヒントから教えてください。
JavaScriptのif文を習ったばかりの中学生です。
「13歳以上15歳以下のときだけメッセージを出す」条件の考え方を、数直線のイメージで説明してください。
答えのコードは最後にお願いします。
