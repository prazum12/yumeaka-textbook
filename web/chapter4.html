Webコース教科書

第4章

自己紹介サイト制作 〜設計から見せ合いまで〜

📅 9/12(土) の講習で使います
💻 使っているPC:

🎯 この章でできるようになること

  • ワイヤーフレーム（設計図）を描いてからサイトを作る、プロと同じ流れを体験できる
  • 第1〜3章で覚えたHTMLとCSSを組み合わせて、自分だけの自己紹介サイトを作れる
  • 作ったサイトの「こだわり」を、人に言葉で説明できる

今日は新しいタグやプロパティは出てきません。
第1〜3章で覚えたことを全部使って、自分のサイトを作る日です。
材料はもうそろっている。
あとは組み立てるだけ！

この章の使い方 … 今日は「読む」より「作る」が主役。
この章はいつもと違って、最初から順番に読む教科書ではなく、詰まったときに開く辞書（リファレンス）です。
4-1で流れをつかんだら、あとは「部品カタログ（4-4）」と「つまずきポイント集（4-6）」を相棒にして、どんどん手を動かそう。

アオイ自由に作っていいと言われると、逆に手が止まる…。

レン最初に紙で場所を決めれば大丈夫。設計→骨組み→装飾→仕上げの順で迷いを減らそう。

4-1. 制作の4ステップ

サイト作りには順番があります。
プロも初心者も同じ、この4ステップ。
いきなり色や飾りから始めると迷子になるので、順番を守るのが完成への近道だよ。

設計に時間をかけすぎないのもコツ。
ワイヤーフレームはさっと描いて、 なるべく早くステップ2〜3の「作る時間」に入ろう。
あとから設計を直したくなったら、直していいんだよ。

4-2. ステップ1〈設計〉ワイヤーフレームを描こう

ワイヤーフレーム … サイトの設計図のこと。
「何を・どの順番で・どう並べるか」を四角と線だけで描いたもの。
色や細かいデザインはまだ決めなくてOK。
プロのWebデザイナーも必ず最初に描きます。

宿題でもう描いてきた人は、それを机の上に置いてステップ2へ！
まだの人は、ここから始めよう。
決めることは3つだけ。

1のせる内容を書き出す（何を）

まずは「ネタ帳」づくり。
自分のサイトにのせたいことを、思いつくまま書き出そう。
定番ネタはこのあたり。

定番ネタ例
なまえ＋ひとことニックネームとキャッチコピー（例:「猫と音ゲーが好きな中2」）
プロフィール学年・部活・性格・最近ハマっていること
好きなものゲーム・音楽・食べものなどをカードで並べて紹介
マイベスト3「好きな曲ベスト3」など、順位のあるもの
リンク集第2章で作ったページや、よく見るサイト
将来の夢なりたい職業、やってみたいこと

🔒 のせてはいけないもの（大事！）

今日作るサイトは自分のPCの中だけで動くけれど、「いつ公開しても大丈夫な作り方」を最初からクセにしよう。

書かない: 本名のフルネーム・学校名・住所・電話番号（名前はニックネームでOK）。

使わない: 顔がはっきりわかる写真。
代わりに自分で描いたイラスト、ペットや風景、持ち物の写真がおすすめ。

2順番を決める（どの順で）

ネタ帳から「これは絶対のせたい」を3〜5個選んで、上から並べる順番を決めます。
いちばん大事なもの（＝自分はだれか）をいちばん上に。
Webページは上から読まれるからね。

3紙に四角で描く（どう並べる）

ノートやいらない紙に、ページの形を描いて中身を四角で置いていきます。
写真は「まる」、文章は「線」で描けば十分。
こんな感じ！

この設計図が、コードを書くとこうなります。
ゴールのイメージを持ってステップ2へ進もう。

4-3. ステップ2〈骨組み〉サイトの土台を作ろう

新しいサイト用に、まっさらなフォルダから始めます。
第1章・第3章でやった手順の復習だよ。

1フォルダを作ってVSCodeで開く

デスクトップに my-site という名前のフォルダを作り、VSCodeの「ファイル」メニュー →「フォルダーを開く」で開きます（第1章 1-3と同じ手順）。

2ファイルとフォルダを用意する

エクスプローラー（VSCodeの左側）で、次の3つを作ります。

フォルダ構成 
my-site
├── index.html   ← ページ本体
├── style.css    ← デザイン
└── img          ← 画像を入れるフォルダ
3土台のコードをコピペする

今日はスピード重視！
土台は打ち込まずコピーでOK。
まず index.html に:

index.html（土台） 
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8">
    <title>じぶん紹介サイト</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <header class="hero">
      <h1>ニックネームのサイト</h1>
      <p>キャッチコピーをここに書く</p>
    </header>

    <main>
      <!-- ここに部品カタログの部品を入れていく -->
    </main>

    <footer class="site-foot">
      <p>つくった人: ニックネーム</p>
    </footer>
  </body>
</html>

続けて style.css に:

style.css（土台） 
body {
  margin: 0;
  font-family: sans-serif;
  background-color: #f1f5f9;
  color: #1e293b;
  line-height: 1.8;
}

main {
  max-width: 700px;
  margin: 0 auto;
  padding: 16px;
}

.hero {
  background-color: #0ea5e9;
  color: #ffffff;
  text-align: center;
  padding: 48px 16px;
}

.hero p {
  color: #e0f2fe;
}

.site-foot {
  text-align: center;
  color: #64748b;
  font-size: 14px;
  padding: 24px;
}
4表示を確認して、自分の言葉に書き換える

index.html の上で右クリック →「Open with Live Server」。
空色のヘッダーが出たら土台は完成！
「ニックネームのサイト」「キャッチコピーをここに書く」を自分の言葉に書き換えて、Ctrl + S⌘ + S で保存しよう（index.html と style.css は別々に保存が必要だよ）。

4-4. ステップ2＆3〈骨組み・装飾〉コピペで使える部品カタログ

ここからが今日のメイン。
ワイヤーフレームと見比べながら、使いたい部品を選んでコピペ → 中身を自分のものに書き換える、をくり返してサイトを組み立てよう。

  • HTMLは <main> の中（<!-- ここに部品カタログの部品を入れていく --> の場所）に上から順に貼る
  • CSSは style.css のいちばん下に貼り足す
  • 全部使わなくてOK。
    順番も自由。
    文章と画像は必ず自分のものに書き換えること！

部品1: プロフィールカード

画像と自己紹介文を横に並べたカード。
写真の代わりに自分で描いたイラストでもOK。

HTML（mainの中へ） 
<section>
  <h2>プロフィール</h2>
  <div class="profile-card">
    <img src="img/icon.png" alt="自分のアイコン" class="profile-img">
    <div>
      <p><strong>なまえ:</strong> ゆめこ（ニックネーム）</p>
      <p><strong>学年:</strong> 中学2年生</p>
      <p><strong>ひとこと:</strong> 音ゲーと猫が好きです。</p>
    </div>
  </div>
</section>
CSS（style.cssの下へ） 
.profile-card {
  display: flex;
  align-items: center;
  gap: 16px;
  background-color: #ffffff;
  border-radius: 12px;
  padding: 20px;
}

.profile-img {
  width: 120px;
  height: 120px;
  border-radius: 50%;
  object-fit: cover;
}
border-radius: 50% … 角を丸くするプロパティ。
50%にすると正方形の画像がまん丸になる。
object-fit: cover は「画像をつぶさず丸に合わせて切り抜く」という意味。
画像がまだ無い人は <img> の行を消しておこう（あとで足せる）。

部品2: 好きなものカード（横並び）

第3章のFlexboxの出番。
カードは2枚でも4枚でもOK、<div class="card"> のかたまりを増減しよう。

HTML（mainの中へ） 
<section>
  <h2>好きなもの</h2>
  <div class="cards">
    <div class="card">
      <h3>ゲーム</h3>
      <p>今は音ゲーにハマっています。毎日30分練習中。</p>
    </div>
    <div class="card">
      <h3>猫</h3>
      <p>うちの猫はキジトラ。ひざの上が定位置です。</p>
    </div>
    <div class="card">
      <h3>カレー</h3>
      <p>横須賀といえばカレー。辛口派です。</p>
    </div>
  </div>
</section>
CSS（style.cssの下へ） 
.cards {
  display: flex;
  gap: 16px;
  flex-wrap: wrap;
}

.card {
  flex: 1;
  min-width: 180px;
  background-color: #ffffff;
  border-radius: 12px;
  padding: 16px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.08);
}

.card h3 {
  margin-top: 0;
  color: #0369a1;
}

部品3: マイベスト3

順位があるから ol（順番ありリスト）を使うよ。
第2章の復習だね。

HTML（mainの中へ） 
<section>
  <h2>好きな曲ベスト3</h2>
  <ol class="best3">
    <li>1位の曲名 — ひとことコメント</li>
    <li>2位の曲名 — ひとことコメント</li>
    <li>3位の曲名 — ひとことコメント</li>
  </ol>
</section>
CSS（style.cssの下へ） 
.best3 li {
  background-color: #ffffff;
  border-radius: 8px;
  padding: 10px 16px;
  margin: 8px 0;
}

部品4: リンク集（ボタン風）

ただの <a> タグも、CSSでボタンに変身。
href の中身は自分のリンクに書き換えよう。

HTML（mainの中へ） 
<section>
  <h2>リンク集</h2>
  <div class="links">
    <a class="btn" href="https://developer.mozilla.org/ja/">MDN（Webの辞書）</a>
    <a class="btn" href="ここに好きなサイトのURL">好きなサイト</a>
  </div>
</section>
CSS（style.cssの下へ） 
.links {
  display: flex;
  gap: 12px;
  flex-wrap: wrap;
}

.btn {
  display: inline-block;
  background-color: #0369a1;
  color: #ffffff;
  text-decoration: none;
  font-weight: bold;
  padding: 10px 20px;
  border-radius: 999px;
}

.btn:hover {
  background-color: #0ea5e9;
}
:hover … 「マウスをのせている間だけ」スタイルを変える書き方。
上の例では、ボタンにマウスをのせると色が明るくなる。
試してみて！

文章がメインの部品。
しめくくりに置くと、サイトがぐっと「自分らしく」なるよ。

  </div>
</section>

  padding: 16px 20px;

4-5. ステップ4〈仕上げ〉＆ 💪 レベルアップ課題

部品を組み終わったら、見せ合いの前に3つだけ見直そう。

  • 仮置きの文字が残っていない？ —「キャッチコピーをここに書く」のままだと恥ずかしい！
  • リンクと画像は全部動く？ — リンクを全部クリック、画像が全部表示されるか確認
  • ウィンドウを狭くしても読める？ — ブラウザの幅を縮めてみよう。
    カードが折り返せば合格（flex-wrap のおかげ）

時間が余った人は、レベルアップ課題に挑戦！

課題1: 配色を「自分の色」に変えよう

style.css の色コードを置き換えるだけで、サイトの雰囲気がガラッと変わります。
まずはこのセットから試してみよう。

セット名背景（body）ヘッダー（.hero）見出し・ボタン
そら（今の色）#f1f5f9#0ea5e9#0369a1
ミント#f0fdf4#16a34a#15803d
サクラ#fdf2f8#ec4899#be185d
レモン#fefce8#f59e0b#a16207
ヒントを見る

置き換える場所は background-color と color の値。
もっと自由に選びたい人は、Googleで「カラーピッカー」と検索すると、色を選んで #○○○○○○ のコードをコピーできる画面が出てくるよ。

課題2: カードに「動き」をつけよう

部品4で使った :hover を、好きなものカード（.card）にも使って、マウスをのせたら見た目が変わるようにしてみよう。

ヒントを見る

style.css の下にこう書き足す:

.card:hover { box-shadow: 0 6px 16px rgba(0, 0, 0, 0.2); }

影の代わりに background-color を変えてもおもしろいよ。

課題3（チャレンジ）: 第2章のページとつなげよう

第2章で作った「好きなもの紹介ページ」（favorite.html）を my-site フォルダにコピーして、リンク集からボタンで飛べるようにしよう。
これで「複数ページのサイト」に進化！

ヒントを見る

①エクスプローラー（Windowsのファイル画面）Finderで yumeaka-web の中の favorite.html（と、使っている画像）を my-site にコピー ②画像は my-site の img フォルダへ ③リンク集に <a class="btn" href="favorite.html">好きなもの紹介</a> を追加。
同じフォルダの中のファイルは、ファイル名だけでリンクできるよ（相対パス！）。

4-6. 🚨 つまずきポイント集

今日は総合演習だから、つまずきポイントも総集編。
上から順に確認していこう。

style.css に書いたのに、見た目が何も変わらない

原因: HTMLとCSSがつながっていない。
<link> の書き忘れ・打ち間違いが一番多い。

直し方: <head> の中に <link rel="stylesheet" href="style.css"> があるか確認。
href のファイル名が実際のファイル名（大文字・小文字まで）と一致しているかもチェック。

一部の部品だけスタイルが効かない

原因: HTMLの class="○○" とCSSの .○○ の名前がズレている。
cards と card の s の有無、大文字・小文字、全角スペースの混入が定番。

直し方: HTML側のclass名をコピーして、CSS側に貼り直すのが確実。
CSSのセレクタの先頭に .（ドット）を忘れていないかも確認しよう。

ある場所から下のCSSが、全部無視される

原因: どこかで }（閉じかっこ）か ;（セミコロン）を忘れていて、そこから先が読めなくなっている。

直し方: 効かなくなり始めた場所のすぐ上のブロックを見る。
{ と } がペアになっているか、各行の終わりに ; があるか。
VSCodeの色分けが突然おかしくなっている場所が犯人だよ。

カードが横に並ばず、縦に積まれる

原因: display: flex; を付ける場所のまちがい。
並べたいもの自身（.card）ではなく、並べたいものを包む親（.cards）に付けるのがルール。

直し方: カード全体を <div class="cards">〜</div> で包んでいるか、CSSで .cards { display: flex; } になっているかを確認。

画像が表示されず、こわれたアイコンが出る

原因: パスかファイル名のまちがい（第2章でもやったね）。

直し方: ①画像を my-site の中の img フォルダに入れたか ②src="img/ファイル名" のつづり・拡張子が1文字ずつ合っているか。
くわしくは第2章の2-7へ。

内容がまん中に寄らず、左に張りついたまま

原因: 土台CSSの main の設定が欠けている。
まん中寄せの正体は max-width（幅の上限）＋ margin: 0 auto;（左右の余白を自動で均等に）のコンビ。

直し方: main { max-width: 700px; margin: 0 auto; } がそろっているか確認。
auto の書き忘れが多いよ。

保存したのに、ブラウザの表示が変わらない

原因: 保存し忘れ。
今日はファイルが2つあるので、index.html と style.css の両方を保存する必要がある。

直し方: タブに「●」が付いているファイルは未保存。
それぞれ Ctrl + S⌘ + S しよう。

4-7. 🤖 AIを味方につけよう
