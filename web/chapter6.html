でも本当にやりたいのは、自分のWebページを動かすことのはず。

問題のテーマはキミの自由だよ。できあがったら、先生やメンターに見せに行こう。

レンまず1問だけ動かそう。動いた仕組みをコピーして増やせば、作品になる。

6-1. DOMとは 〜HTMLとJavaScriptの橋渡し〜

ブラウザはHTMLファイルを読みこむと、ページの部品（見出し・段落・ボタンなど）を ツリー（木）の形の一覧表にして持っています。
これを DOM（ドム）と呼びます。
JavaScriptはこのDOMを通して部品をつかまえて、「文字を変えて」「色を変えて」と命令できるんです。

DOM（ドム） … Document Object Model の略。
ブラウザが持っている「ページの部品一覧表」。
JavaScriptからHTMLをさわるときの窓口になる。

6-2. 部品を「つかまえて」変えてみよう

DOMから部品をつかまえるいちばん簡単な方法が document.getElementById() です。
第2章で学んだ「属性」を使って、HTML側の部品に id（名前）をつけておき、その名前で呼び出します。

新しいファイル dom-test.html を作って、ためしてみよう。

dom-test.html 
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8">
    <title>DOMの練習</title>
  </head>
  <body>
    <h1 id="aisatsu">こんにちは！</h1>

    <script>
      let aisatsu = document.getElementById("aisatsu");
      aisatsu.textContent = "こんばんは！";
      aisatsu.style.color = "orange";
    </script>
  </body>
</html>

ブラウザで開くと、HTMLには「こんにちは！」
と書いたのに、画面にはオレンジ色の「こんばんは！」が表示されます。
JavaScriptがあとから部品をつかまえて書き換えたからです。
これがDOM操作！

書き方意味
document.getElementById("名前")「ページ（document）の中から、このidの部品を取ってきて」という命令
部品.textContent = "文字"部品の中の文字を書き換える
部品.style.color = "色"部品の文字色を変える（CSSの color と同じ）
部品.style.backgroundColor = "色"背景色を変える。
CSSの background-color はJSではハイフンなしで次を大文字にする
id属性 … 部品につける「自分だけの名前」。
同じidはページに1回しか使えない。
JSからつかまえたい部品に付けよう。
つづりはHTML側とJS側で1文字も違わず一致させること！

🚨 先に知っておこう: <script> は body の最後に書く

ブラウザはHTMLを上から順に読みます。
部品より先にJSを書くと「そんな部品はまだ無いよ（null）」というエラーに。
</body> の直前に書くのが安全です。

6-3. ボタンをクリックしたら動かす 〜イベント〜

6-2のコードは、ページを開いた瞬間に1回動いて終わりでした。
「ボタンを押したときに動く」ようにするには、イベントという仕組みを使います。

イベント … ページの上で起きる「できごと」のこと。
クリック（click）のほかに、マウスが乗った（mouseover）、キーを押した（keydown）などがある。
関数（かんすう） … 命令をひとまとめにした「実行の予約セット」。
function () { ... } の { } の中に、やってほしい命令を書く。

dom-test.html の <body> の中身を、次のように書き換えてためそう。

dom-test.html（bodyの中身） 
<button id="botan">押してみて</button>
<p id="message">まだ押されていない</p>

<script>
  let botan = document.getElementById("botan");
  let message = document.getElementById("message");

  botan.addEventListener("click", function () {
    message.textContent = "押してくれてありがとう！";
    message.style.color = "deeppink";
  });
</script>

ボタンを押した瞬間、文字が変わったら成功！
実はこの教科書の「コピー」ボタンも、まったく同じ addEventListener で動いています。
キミはもう、この教科書と同じ技術を使えるようになったってこと。

6-4. 配列 〜クイズの材料箱〜

クイズに必要な残りの材料は「問題や選択肢をまとめて覚えておく箱」。
第5章の変数 let は値を1つしか覚えられなかったけど、配列を使うとまとめて覚えられます。

配列: 値をまとめて入れる箱

配列の例（コンソールでためせる） 
let sentakushi = ["富士山", "高尾山", "エベレスト"];

console.log(sentakushi[0]);      // → 富士山（番号は0から始まる！）
console.log(sentakushi[2]);      // → エベレスト
console.log(sentakushi.length);  // → 3（入っている個数）
配列（はいれつ） … [ ] にカンマ区切りで値を並べた「まとめ箱」。
箱[番号] で取り出す。
番号は1ではなく0から始まるのがプログラミングの伝統なので注意！

配列の中に配列 〜クイズ1問をパックする〜

配列には文字だけでなく、数も、そして配列そのものも入れられます。
「問題文・選択肢3つ・正解の番号」を1つの配列にパックすれば、クイズ1問が1行で書ける！

1問パックの例（コンソールでためせる） 
let quiz = ["日本でいちばん高い山は？", "富士山", "高尾山", "エベレスト", 1];

console.log(quiz[0]);   // → 日本でいちばん高い山は？（問題文）
console.log(quiz[1]);   // → 富士山（選択肢その1）
console.log(quiz[4]);   // → 1（正解の番号）

そして「1問パック」をさらに配列に並べれば、問題集のできあがり。
これがクイズの心臓部だよ。

問題集（クイズの心臓部） 
let quizList = [
  ["日本でいちばん高い山は？", "富士山", "高尾山", "エベレスト", 1],
  ["1週間は全部で何日？", "5日", "6日", "7日", 3],
];

console.log(quizList.length);   // → 2（問題の数）
console.log(quizList[1][0]);    // → 1週間は全部で何日？（2問目の問題文）
配列の中の配列 … quizList[1][0] は「quizListの1番のパックの中の、0番」という意味。
番号を2回重ねて、奥の値を取り出せる。

問題の数を数えるのに quizList.length を使っているのがポイント。
問題を増やしても、プログラムの他の部分は直さなくていいんです。

6-5. ミニ作品「3択クイズ」を完成させよう

ここまでの材料を全部合体させて、3択クイズを作ります。
まずは完成イメージから。

今日は制作と発表の時間をたっぷり取りたいので、土台のコードはテンプレートとして用意しました。
まずはコピペで動かしてOK！
そのあと問題を入れかえて「自分の作品」に育てるのが今日のメインイベントです。

1quiz.html を作る

VSCodeで自己紹介サイトのフォルダを開き、新しいファイル quiz.html を作ります（あとで組み込むのが楽になるよ）。

2テンプレートを貼りつける

下のコードをコピーして貼りつけ、Ctrl + S⌘ + S で保存しよう。

quiz.html（テンプレート） 
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8">
    <title>3択クイズ</title>
    <style>
      body {
        font-family: sans-serif;
        text-align: center;
        background-color: #eff6ff;   /* ★カスタム1: 背景の色 */
      }
      h1 {
        color: #1d4ed8;              /* ★カスタム2: タイトルの色 */
      }
      #mondai {
        font-size: 24px;
        font-weight: bold;
        margin: 30px 10px;
      }
      button {
        display: block;
        margin: 12px auto;
        width: 280px;
        font-size: 18px;
        padding: 12px;
        border: none;
        border-radius: 999px;
        background-color: #2563eb;   /* ★カスタム3: ボタンの色 */
        color: #ffffff;
        cursor: pointer;
      }
      #handan {
        font-size: 28px;
        font-weight: bold;
        height: 42px;
      }
    </style>
  </head>
  <body>
    <h1>なんでも3択クイズ</h1>  <!-- ★カスタム4: クイズのタイトル -->
    <p id="mondai"></p>
    <button id="botan1"></button>
    <button id="botan2"></button>
    <button id="botan3"></button>
    <p id="handan"></p>

    <script>
      // ★カスタム5: 問題リスト。[問題文, 選択肢1, 選択肢2, 選択肢3, 正解の番号] で1問
      let quizList = [
        ["日本でいちばん高い山は？", "富士山", "高尾山", "エベレスト", 1],
        ["1週間は全部で何日？", "5日", "6日", "7日", 3],
        ["大きな見出しを作るHTMLタグは？", "<p>", "<h1>", "<a>", 2],
      ];

      let bango = 0;      // いま何問目か（0から数える）
      let seikaisu = 0;   // 正解した数

      let mondai = document.getElementById("mondai");
      let handan = document.getElementById("handan");
      let botan1 = document.getElementById("botan1");
      let botan2 = document.getElementById("botan2");
      let botan3 = document.getElementById("botan3");

      // いまの問題をボタンに表示する
      function hyoji() {
        mondai.textContent = "Q" + (bango + 1) + ". " + quizList[bango][0];
        botan1.textContent = quizList[bango][1];
        botan2.textContent = quizList[bango][2];
        botan3.textContent = quizList[bango][3];
      }

      // 答え合わせをして、次へ進む
      function kotaeru(erabinda) {
        if (erabinda === quizList[bango][4]) {
          handan.textContent = "⭕ 正解！";
          handan.style.color = "green";
          seikaisu = seikaisu + 1;
        } else {
          handan.textContent = "❌ ざんねん…";
          handan.style.color = "red";
        }
        bango = bango + 1;
        if (bango < quizList.length) {
          hyoji();
        } else {
          mondai.textContent = "おわり！ " + quizList.length + "問中 " + seikaisu + "問正解！";
          botan1.style.display = "none";
          botan2.style.display = "none";
          botan3.style.display = "none";
        }
      }

      botan1.addEventListener("click", function () { kotaeru(1); });
      botan2.addEventListener("click", function () { kotaeru(2); });
      botan3.addEventListener("click", function () { kotaeru(3); });

      hyoji();   // 最初の問題を表示してスタート
    </script>
  </body>
</html>
3動かして、コードを読む

右クリック →「Open with Live Server」で開いて、全問答えてみよう。
最後にスコアが出たら成功。
動いたら、コードを上から読んで「ほとんど知ってる文法だ」と確認してみて。
魔法は1行もないよ。

このテンプレートは、実はこれまで学んだことの総まとめでできています。

テンプレートの部分学んだ場所
HTMLの骨組み・タグ・id属性第1〜2章
<style> の中の色や形の指定第3〜4章
変数 let・if文・===第5章
DOM・イベント・配列この章

プログラムの流れはこう。
hyoji() がいまの問題をボタンに表示 → ボタンが押されたら kotaeru() が答え合わせをして bango を1進める → 問題が残っていればまた hyoji()、なくなったらスコア発表。
初登場の書き方は2つだけです。

名前つき関数 … function hyoji() { ... } と書くと、命令の束に「hyoji」という名前がつく。
hyoji(); と呼ぶたびに束が実行される。
「何度も使う処理」をまとめるのに便利。
引数（ひきすう） … 関数にわたす「メモ」。
kotaeru(1) と呼ぶと、function kotaeru(erabinda) の erabinda に 1 が入る。
このメモで「どのボタンが押されたか」を関数に伝えている。

6-6. 💪 やってみよう 〜クイズを「自分の作品」にする〜

ここからが本番。
テンプレートのままでは、まだ「キミの作品」じゃない。
★カスタム1〜5を目印に、どんどん改造しよう。

課題1: 問題を自分のテーマで作る

テンプレートの問題を全部消して、自分の好きなテーマの問題に入れかえよう。
推し・部活・ゲーム・マンガ・生き物、なんでもOK。
5問以上作ると、遊びごたえのあるクイズになるよ。
友だちが「えっ、どっち！
?」と迷う選択肢を作れたら勝ち。

ヒントを見る

★カスタム5の配列に、["問題文", "選択肢1", "選択肢2", "選択肢3", 正解の番号], の行を追加するだけ。
判定は quizList.length を使っているから、増やしても他の行は直さなくていい。
正解の番号（1〜3）の書き忘れに注意！

課題2: 見た目を自分のテーマに変える

★カスタム1〜4を変えて、クイズのテーマに合う見た目にしよう。
タイトル・背景色・ボタンの色がそろうと作品らしくなる。

ヒントを見る

色の名前（pink など）でも、第3章で使ったカラーコード（#f59e0b など）でもOK。
タイトルの文字（なんでも3択クイズ）を自分のテーマ名に変えるのも忘れずに。

課題3（チャレンジ）: 全問正解のときだけ特別なメッセージ

第5章のif文の出番！
全問正解したときだけ「全問正解！
キミは天才！」
のような特別なメッセージが出るようにしてみよう。

ヒントを見る

「全問正解」は、seikaisu と quizList.length が同じ数になったとき。
テンプレートの「おわり！
…」の行を、このif文にまるごと置きかえよう。

考え方の例 
if (seikaisu === quizList.length) {
  mondai.textContent = "全問正解！ キミは天才！";
} else {
  mondai.textContent = "おわり！ " + quizList.length + "問中 " + seikaisu + "問正解！";
}

余裕があれば、全問正解のときに mondai.style.color で文字色も変えてみよう。

話すこと例

  • 苦労した話こそおもしろい。エラーと戦った話は、聞いている先生・メンターも興味しんしん

    時間が余ったら、近くの友だちに見せ合うのもOK（発表会ではなく気軽に）

6-9. 🚨 つまずきポイント集

ボタンを押しても何も起こらない

原因: HTML側の id="..." とJS側の getElementById("...") のつづりが一致していない（この章で一番多いエラー！）。

直し方: 開発者ツール（F12⌘ + option + I）のコンソールを開こう。
Cannot set properties of null のような赤いエラーが出ていたら、id不一致のサイン。
大文字・小文字の違いもアウトだよ。
idが合っているのに出るときは、6-2の「<script> は body の最後に書く」も確認しよう。

「... is not a function」というエラーが出る

原因: 命令の大文字・小文字ミス。
getElementByID（最後が大文字）や addeventlistener は別物あつかいになる。

直し方: getElementById・addEventListener と1文字ずつ見比べる。
VSCodeの補完（途中まで打つと出る候補）を使うとミスが減るよ。

問題文やボタンに「undefined」と表示される

原因: 問題の配列が「5個セット」になっていない。
選択肢のあいだのカンマを忘れたり、最後の正解番号を書き忘れたりすると、配列に「ない番号」を取りに行って undefined になる。

直し方: quizList の各行が [問題文, 選択肢, 選択肢, 選択肢, 正解の番号] の5個そろっているか、カンマの位置と一緒に確認しよう。

正解のボタンを押したのに「❌」になる

原因: 正解番号のズレ。
番号を0から数えてしまった、または選択肢の順番を入れかえたのに正解番号を直し忘れた、が定番。

直し方: 正解の番号は上のボタンから1・2・3（ここは0からじゃない）。
問題を編集したら、その行の正解番号もセットで見直すクセをつけよう。

「Unexpected token」エラー／とにかく動かない

原因: 全角文字の混入が定番。
日本語入力のままコードを打つと、全角スペースや全角の "・（） が入りやすい。
問題文を日本語で入力したあとは特に起きやすい！

直し方: エラーに出ている行番号のあたりを、半角で打ち直す。
" や , や [ ] などの記号は半角、" " の中の問題文は日本語でOK。
VSCodeの色分けがそこだけ変なら、それがヒント。

6-10. 🤖 AIを味方につけよう

この章でのAIの使い所

✅ 良い使い方 — まず15分自分で考えてから

  • エラーメッセージの翻訳: コンソールの赤い英語を貼って「どういう意味？」
    と聞く
  • 改造の相談: 「こう変えたい。
    今のコードはこれ。
    どこを変えればいい？」
    と自分のコードを添えて聞く
  • コードの意味の確認: テンプレートの中で意味があいまいな行を説明してもらう

そのままコピペで使えるプロンプト（AIへの指示文）の例:

JavaScriptを勉強中の中学生です。
3択クイズのボタンを押しても何も起きません。
コンソールには「Uncaught TypeError: Cannot set properties of null」と出ています。
コードはこれです:（自分のコードを貼る）。
いきなり答えではなく、原因の見つけ方を順番に教えてください。
この3択クイズを「問題がランダムな順番で出る」ように改造したいです。
今のコードはこれです:（自分のコードを貼る）。
変更する行だけを示して、なぜそれで動くのかも説明してください。
