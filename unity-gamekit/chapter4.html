第4章

敵とダメージのしくみ 〜ChomperとSpitterを攻略せよ〜

📅 第4回の講習で使います（日付が決まったらここに記載）
💻 使っているPC:

🎯 この章でできるようになること

  • 2種類の敵（ChomperとSpitter）をステージに配置できる
  • Enemy Behaviourコンポーネントで、敵の速さや視野（View FOV）を調整できる
  • 数値をドラッグで変える「プロのちょい技」が使える
  • PushableBoxにDamagerコンポーネントを追加して、「箱を落として敵を倒す」しかけを作れる

動く床と扉ができた前回のステージ。
でも、まだ何かが足りない…そう、敵だ。
今日はステージにモンスターを放って、さらに「ダメージのしくみ」を自分の手で組み立てます。
最後には、箱を頭上から落として敵を倒すトラップが完成するよ。

4-1. 敵を配置して、動きを調整しよう

2D Game Kitには、作成済みの敵が2種類入っています。
置き場所は Prefabs > Enemies。前回のしかけと同じ「プレハブ」だから、ドラッグするだけで動き出すよ。

敵特徴
Chomper（チョンパー）地面を歩き回って、近づくとかみついてくる。接近戦タイプ
Spitter（スピッター）プレイヤーを見つけると、酸のたまを吐いて撃ってくる。遠距離タイプ
1Prefabs > Enemies からChomperをドラッグ

Projectウィンドウで「Prefabs」→「Enemies」の順にクリックして、Chomper をシーンビューへドラッグしよう。
置いたら▶で再生してみて。もう歩き回ってるはず。
武器が出ないときは F12 で有効化、O や K で攻撃できるよ。

コンポーネント … ゲームオブジェクトに取り付ける「機能の部品」のこと。
Chomperが歩いたり噛みついたりするのは「Enemy Behaviour」というコンポーネントのおかげ。
インスペクターに並んでいる1つ1つの箱が、ぜんぶコンポーネントだよ。
部品を付けたり外したりして機能を組み立てるのが、Unityの基本スタイルなんだ。
2Enemy Behaviourで速さと視野を調整する

Chomperを選択して、インスペクターの Enemy Behaviour コンポーネントを見てみよう。
ここが敵の「性格」を決める調整パネルだ。

  • Speed … 動きの速さ。上げるとおそろしい敵になる
  • View FOV … 視野の広さ（FOV＝Field Of View、視野角）。シーンビューに扇形で表示される
  • View Direction … 視野の向き。どっちを見張っているか

いろいろ変えて再生し、動きのちがいを確かめよう。
ちなみにChomperとSpitterは同じEnemy Behaviourコンポーネントを持っていて、設定値がちがうだけなんだ。

3プロのちょい技: プロパティは「ドラッグ」で変えられる

数値はキーボードで打ちこんでもいいけど、もっと速い方法があります。
プロパティの名前（SpeedやView FOVの文字）の上で左クリックしたまま左右にドラッグしてみて。
数値がスルスル変わる！
微調整しながら結果を見たいときに最強のテクニックなので、今日から使っていこう。

4-2. 箱を落としてSpitterを倒そう

ここからがこの章のメインイベント、ダメージのしくみ作りです。
ちょっとかわいそうだけど…Spitterの頭上に箱を落として倒します。
完成するとこんな感じ。

1高低差のあるステージを作る

まず、Ellenが高い所、敵が低い所という配置になるよう、タイルで崖のある地形を作ろう。
（タイルを塗るときはTile Palette、塗り終わったら W で移動ツール。もう手なれたもんだね）

2Spitterを崖の下に置いて、View Distanceを下げる
  • Prefabs > Enemies から Spitter を1体シーンビューへドラッグ
  • ステージの下の段、崖の面の横に配置する
  • Spitterを選択したまま、インスペクターで Enemy Behaviour (Script) を表示
  • View Distance（見える距離）の数値を下げる

View Distanceを下げるのは、テスト中にSpitterがこちらを見つけてすぐ撃ってこないようにするため。
さっき覚えた「文字の上をドラッグ」で下げてみよう。

3PushableBoxを置いて、Damagerを追加する
  • Prefabs > Interactables から PushableBox（押せる箱）を崖の上へドラッグ
  • ヒエラルキーで PushableBox を選び、インスペクターの一番下にある「Add Component」ボタンをクリック
  • 検索ボックスに「Damage」と入力
  • 出てきた「Damager」をクリックして追加

Damagerは「さわった相手にダメージを与える」コンポーネント。
ダメージを受けられる側（SpitterやChomper）は「Damageable」というコンポーネントを持っていて、DamagerとDamageableが出会うとダメージが発生する——これがこのキットのダメージのしくみだ。

4緑のボックスを、箱にピッタリ重ねる

図4-7を見ると、緑のボックスが箱からズレているのがわかるかな？
このままだと箱を落としても、緑ゾーンが当たらないのでダメージゼロ。
緑のボックスをPushableBoxとだいたい同じ位置・サイズに合わせよう。方法は2つ。

方法1: シーンビューで直接ドラッグ
緑の輪郭線上にある緑の点をドラッグして、箱に重なるようにサイズと位置を調整。

方法2: インスペクターのOffsetとSizeで調整
Damagerコンポーネントの Offset（位置のズレ）と Size（大きさ）を変える。
ここでも「X・Yの文字の上を左右にドラッグ」が最速だよ。

5HittableLayersを「Enemy」にする

最後に「誰にダメージを与えるか」を決めます。
Unityでは、オブジェクトをレイヤーというグループで分類しているんだ。

  • PushableBox を選択
  • インスペクターで Damager コンポーネントを表示
  • 「HittableLayers」のドロップダウンで「Enemy」を選ぶ

これで、この箱は「Enemy」レイヤーのオブジェクト（SpitterもChomperも）にダメージを与えるようになった。

レイヤー（Layer） … オブジェクトを分類する「グループ分けのラベル」。
「Enemy（敵チーム）」「Player（プレイヤーチーム）」のように分けておくと、「敵チームだけにダメージ」「プレイヤーチームだけに反応」といったチーム単位の指定ができる。
出席番号じゃなくて「クラス全員に配る」と言えるのが便利なところ。
6再生して、箱を押して落とす！

▶（再生）を押したら、Ellenで箱をグイグイ押して崖から落とそう。
Spitterの頭上に命中したら…撃破！ 図4-4の完成イメージ通りになったかな？
うまくいったら停止して保存（Ctrl + S⌘ + S）。

💡 余裕があったら、他の敵にも箱をぶつけてみよう。
なんならEllenに当てるとどうなるかも実験してみて（HittableLayersを変えると…？）。

4-3. 💪 やってみよう

課題1: 背後からしのび寄れるChomperを作ろう

Chomperを1体置いて、View FOVとView Directionを調整し、「正面から行くと見つかるけど、背後からなら気づかれずに近づける」見張り役にしよう。
再生して、実際に背後から近づいて攻撃できることを確認してね。

ヒントを見る

シーンビューに表示される扇形が「敵の見えている範囲」。
View FOVをせまくして、View Directionを片側（正面方向）に向ければ、背中側は死角になる。
図4-3の設定例が大ヒントだよ。

課題2: 「箱落としトラップ」を自分のステージに組みこもう

4-2で作った箱落としを、自分のステージの一部として組みこもう。
「下にSpitterがいて通れない → 崖の上の箱に気づく → 落として道を開く」という謎解きの流れになっていたら満点。

ヒントを見る

Spitterを「通り道のド真ん中」に置くと、倒さないと進めない状況が作れる。
Damagerの緑ボックス合わせとHittableLayers=Enemyを忘れずに。
箱が押しにくいときは、崖のふちを平らにしてあげよう。

課題3（チャレンジ）: 敵の性格をデザインして「ボス地帯」を作ろう

Enemy Behaviourの数値だけで、性格のちがう敵を2体作ってみよう。
例:「足はおそいが視野がとても広い見張り番」＋「視野はせまいが超高速のアタッカー」。
2体を同じ部屋に置いて、となりの人にクリアしてもらえたら大成功。

ヒントを見る

SpeedとView FOV・View Distanceの組み合わせで性格が決まる。
「文字の上をドラッグ」で数値をいじりながら、再生→調整をくり返そう。
強すぎたら数値を下げる勇気も、ゲームデザイナーの実力のうち。

4-4. 🚨 つまずきポイント集

敵を置いたのに動かない・攻撃してこない

原因: 再生していない。またはEllenが敵の視野（扇形）の外にいる。

直し方: ▶で再生してから確認しよう。
敵はView FOVとView Distanceの範囲にEllenが入ってはじめて反応するよ。
シーンビューで敵を選択すると視野の扇形が見える。

攻撃したいのに武器が出ない

原因: 武器が無効になっている。

直し方: 再生中に F12 キーを押すと武器の有効・無効を切り替えられる。
攻撃は O か K キーだよ。

テスト中、Spitterにすぐ撃たれてしまう

原因: SpitterのView Distance（見える距離）が長いまま。

直し方: Spitterを選んで、Enemy Behaviour (Script) の View Distance を下げよう。
しかけが完成したら、好みの強さに戻せばOK。

箱をSpitterに落としたのにダメージが入らない

原因1: Damagerの緑のコリジョンボックスが箱とズレている。
→ 4-2の手順4で、緑ボックスを箱にピッタリ重ねる。

原因2: HittableLayersが設定されていない（Nothingのまま）。
→ Damagerの HittableLayers を「Enemy」にする。

原因3: DamagerをPushableBoxではなく別のオブジェクトに追加してしまった。
→ ヒエラルキーで PushableBox を選んでからAdd Componentしたか確認。

Add Componentで「Damager」が見つからない

原因: 検索ボックスの入力ミス。

直し方: 検索ボックスに半角で「Damage」と入力すると候補に出てくる。
日本語入力モードになっていないかもチェックしよう。

箱が押せない・崖から落ちてくれない

原因: 崖のふちに段差やタイルの出っぱりがあって、箱がひっかかっている。

直し方: 箱の通り道を平らにタイルを塗り直そう。
それでもダメなら、箱の初期位置を崖のふちギリギリに移動しておくと楽に落とせるよ。

💡 15分たっても直らないときは、次の「AIを味方につけよう」の方法で相談してみよう。

4-5. 🤖 AIを味方につけよう

今日は「コンポーネント」と「レイヤー」という、Unityのド真ん中の概念が出てきたね。
しくみの「なぜ？」を深掘りする質問は、AIがいちばん力を発揮するところだよ。

この章でのAIの使い所

✅ 良い使い方 — 自分で15分考えてわからなかったら聞く

  • ダメージが入らない: 「何に」「どのコンポーネントを付けて」「どう設定したか」を具体的に伝える
  • しくみの理解: 「DamagerとDamageableはどう協力してるの？」のような概念の質問はAIの得意分野

そのままコピペで使えるプロンプト（AIへの指示文）の例:

Unity 2022.3の2D Game Kitで、PushableBoxにDamagerコンポーネントを追加して、箱を落としてSpitterを倒すしかけを作っている中学生です。
HittableLayersはEnemyにしました。でも箱がSpitterに当たってもダメージが入りません。
確認すべきことを、順番に1つずつ教えてください。
Unityの「レイヤー」という仕組みを、中学生にわかるようにたとえ話で説明してください。
「ダメージを与える相手をレイヤーで指定する」のが、なぜ1体ずつ指定するより便利なのかも教えてください。

🚫 やめておこう

  • 敵の強さの数値を丸ごとAIに決めてもらう（テストプレイして決めるのはキミの仕事。相談はOK、丸投げはNG）
  • AIの答えを動かして確かめずに信じる（必ず自分の画面で再生して確認）
