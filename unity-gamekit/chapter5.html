第5章

世界を飾ってワープさせよう 〜装飾とテレポート〜

📅 第5回の講習で使います（日付が決まったらここに記載）
💻 使っているPC:

🎯 この章でできるようになること

  • Art > Sprites > Environment の装飾スプライトで、ステージを自由に飾れる
  • サンプルゲームで見つけた「あれ何？」を、Projectウィンドウの名前検索で探せる
  • TransitionStartとTransitionEndで、同じシーン内のワープ（テレポート）を作れる
  • 「触れたらワープ」と「Eキーでワープ」を使い分けられる

動くしかけも敵もそろったキミのステージ。
でも、ちょっと殺風景じゃない？
今日は世界を飾りつけて、仕上げにワープ装置で空間をつなげます。
ステージが一気に「冒険の世界」に化ける回だよ。

5-1. 装飾でステージを「映え」させよう

2D Game Kitには、装飾用のスプライト（絵の素材）がたっぷり入っています。
しかもぜんぶ、サンプルゲーム『The Explorer』の中で実際に使われているものだ。

1Art > Sprites > Environment を開く

Projectウィンドウで「Art」→「Sprites」→「Environment」の順にクリック。
草、木、岩、遺跡のかけら…装飾アイテムがずらり。
サブフォルダーの小さな矢印も展開して、各カテゴリーの中のスプライトもぜひ見てみよう。

2好きなスプライトをシーンへドラッグ

気に入ったスプライトをシーンビューへドラッグするだけで配置完了。
位置の調整は、おなじみの移動ツール（Wキー）でどうぞ。
地面に草を生やす、背景に木を立てる、ゴール前に目立つ像を置く…
置くだけでステージの雰囲気がガラッと変わるよ。

⚠️ 装飾はあくまで「絵」。当たり判定（コライダー）はないので、乗ったりぶつかったりはできません。
足場にしたい場所は、今まで通りタイルで作ろう。

3小ワザ: 「あれ何？」は名前で検索

サンプルゲームの中で目を引くものを見つけたら、それをシーンビューでクリックして、ヒエラルキーで名前を確認。
その名前をProjectウィンドウ上部の検索ボックスに入力すれば、同じ部品がすぐ見つかる！
「見つけて、調べて、自分のステージで使う」——プロもやっている探し方だよ。

5-2. 同じシーン内でワープ装置を作ろう

次はこの章のメインイベント、プレイヤーのテレポートです。
プレイヤーをワープさせるには、遷移（Transition）という設定を作ります。
同じシーン内のワープに使うのは、この2つ。

ゲームオブジェクト役割
TransitionStartワープの入口。プレハブをそのまま置く
TransitionEndワープの出口。同じプレハブをもう1つ置いて、名前を変えたもの
遷移（Transition） … 場面やプレイヤーの位置が「切り替わる」こと。
同じシーン内のワープも、別のステージへの移動も、Unity的にはぜんぶ「遷移」。
入口と出口をセットで用意するのが基本だよ。
1TransitionStart（入口）を置く

Projectウィンドウで「Prefabs」→「SceneControl」の順にクリックして、TransitionStart プレハブをシーンビューへドラッグ。
Ellenが歩いたときに緑のボックス（コライダー）に触れる位置に置くのがポイントだよ。

2もう1つ置いて「TransitionEnd」に改名

同じ SceneControl フォルダーから、TransitionStart をもう1つシーンビューへドラッグ。
そして、インスペクター上部の名前欄で名前を「TransitionEnd」に変更しよう。
これがワープの出口になります。ワープして出てきたい場所に置いてね。

3入口の設定①: 「誰を」「どんなワープで」

ヒエラルキーで TransitionStart を選択して、インスペクターの TransitionPoint コンポーネントを見つけよう。

  • ヒエラルキーから Ellen を、「Transitioning Game Object」のスロットへドラッグ（＝ワープするのはEllenだけ、という意味）
  • 「Transition Type」を「Same Scene」に設定（＝同じシーン内でのワープ）
4入口の設定②: 「どこへ」「いつ」

同じ TransitionPoint コンポーネントで、続けて設定します。

  • ヒエラルキーから TransitionEnd を、「Destination Transform」のスロットへドラッグ（＝出口はここ）
  • 「Transition When」を「On Trigger Enter」に設定

「Transition When」はワープが発動するタイミング。2種類から選べるよ。

設定発動のしかた
On Trigger Enter緑のボックスに触れた瞬間にワープ。自動ドア型。キー操作では発動しない
Interact Pressed緑のボックスに触れた状態でインタラクトキー（E）を押したときだけワープ。エレベーターのボタン型
5再生してワープ！

▶（再生）を押して、Ellenを入口まで歩かせよう。
緑のボックスに触れたら…シュンッ！ TransitionEndの位置に瞬間移動！
うまくいったら停止して保存（Ctrl + S⌘ + S）。
余裕があったら「Transition When」を「Interact Pressed」に変えて、Eキーで発動するちがいも体験してみて。

5-3. 🚀 発展: 別のシーンへワープしよう

ワープは同じシーン内だけじゃなく、別のシーン（別のステージ）へ飛ぶこともできます。
ここは早く終わった人向けの発展コーナー。挑戦したい人は開いてみよう。

🚀 発展: 自分のステージからZone1へワープさせてみたい人はここを開こう

別のシーンへ飛ぶときは、出口側のプレハブが変わります。

  • TransitionStart … 入口。5-2と同じプレハブ。プレイヤーを目的地へ「転送」する側
  • TransitionDestination … 別のシーン側でプレイヤーを「受け取る」専用プレハブ（Prefabs > SceneControl の中）

手順A: 出口（別のシーン側）を設定する

  • 行き先のシーンを開いて、Prefabs > SceneControl > TransitionDestination をワープ先にしたい位置に配置
  • その Scene Transition Destination コンポーネントで、「Destination Tag」に好きなアルファベットを選ぶ（同じシーン内でかぶっていなければどの文字でもOK）
  • ヒエラルキーから Ellen を「Transitioning Game Object」のスロットへドラッグ
  • メニューの File > Build Settings を開いて「Add Open Scenes」をクリック（行き先のシーンをゲームに登録する大事な一手）

手順B: 入口（TransitionStart側）を設定する
設定は5-2とほぼ同じで、次の3点だけ変わります。

  • 「Transition Type」を「Different Zone」に設定
  • 「New Scene Name」を行き先のシーン名に設定
  • 「Transition Destination Tag」を、手順Aで選んだアルファベットに合わせる

例: サンプルゲームの最初のステージへ飛んでみよう
自分のステージのTransitionStartを、こう設定してみて。

  • 「Transition Type」＝「Different Zone」
  • 「New Scene Name」＝「Zone1」
  • 「Transition Destination」＝「A」

▶で再生して入口まで歩けば、Zone1の冒頭にテレポートされるはず！
自分のステージとサンプルゲームの世界がつながる、ちょっと感動の瞬間だよ。

5-4. 💪 やってみよう

課題1: 隠し部屋ワープを作ろう

ステージのすみにタイルで小さな宝部屋を作り、本編からは行けない場所に置こう。
そして、見つけにくい場所（装飾の陰、行き止まりの奥など）にTransitionStartを置いて、宝部屋へワープできるようにしよう。
部屋から戻ってくるワープも作れたら満点！

ヒントを見る

戻りのワープは、逆向きのTransitionStart／End をもう1組作ればOK。
2組に増えると混乱しやすいので、名前を ToRoom・ToField のようにわかりやすく変えておこう。

課題2: 「Eキーで乗るエレベーター」風ワープを作ろう

「Transition When」を「Interact Pressed」にしたワープを1つ作って、触れただけでは飛ばず、Eキーを押したときだけ上の階（高い足場）へ移動できるようにしよう。
ワープ地点のまわりを装飾スプライトで飾って、「乗り物っぽさ」を演出できたらすばらしい。

ヒントを見る

On Trigger Enterだと、通りかかっただけで飛んでしまう。
Interact Pressedなら「ワープしたい人だけがワープ」できるので、通路の途中に置いても事故らないよ。

課題3（チャレンジ）: 装飾で「物語」を演出しよう

装飾スプライトを使って、ステージに雰囲気の変化を作ろう。
例:「草原からスタートして、奥へ進むと遺跡の入口が見えてくる」。
できたら、となりの人に自分のステージを見てもらって、「どんな場所の冒険か」を当ててもらおう。伝わったら大成功！

ヒントを見る

迷ったらサンプルゲームを歩いて研究。
「いいな」と思った飾りは、5-1の小ワザ（名前を確認してProjectウィンドウで検索）ですぐ見つけられるよ。

5-5. 🚨 つまずきポイント集

入口に触れてもワープしない

原因1: 「Transitioning Game Object」が空のまま。
→ ヒエラルキーから Ellen をスロットへドラッグ。

原因2: 「Destination Transform」が空のまま。
→ TransitionEnd をスロットへドラッグ。

原因3: 「Transition When」が「Interact Pressed」になっている。
→ 入口の上で E キーを押すか、「On Trigger Enter」に変えよう。

ワープはするけど、変な場所に飛ばされる

原因: TransitionEnd（出口）がその場所にある。

直し方: ヒエラルキーで TransitionEnd を選び、移動ツール（W）で出てきてほしい場所に置き直そう。
地面の中に置くと埋まっちゃうので、地面より少し上に置くのがコツ。

どっちが入口でどっちが出口か、わからなくなった

原因: 2つとも見た目がそっくり（元は同じプレハブだからね）。

直し方: ヒエラルキーで名前を確認しよう。改名し忘れていたら、インスペクター上部の名前欄で「TransitionEnd」に変更。
ワープが増えてきたら Warp1_Start／Warp1_End のようにセットで名前を付けると迷子にならないよ。

装飾スプライトに乗ろうとしたら、すり抜けた

原因: 装飾はあくまで「絵」で、当たり判定（コライダー）を持っていない。

直し方: それで正常！ 乗れる足場が欲しい場所は、タイルで作ろう。
「見た目は装飾、足場はタイル」と役割分担するときれいに作れる。

（発展）別のシーンへのワープで、何も起きない・エラーが出る

原因1: 行き先のシーンがBuild Settingsに登録されていない。
→ 行き先のシーンを開いた状態で File > Build Settings →「Add Open Scenes」。

原因2: 「New Scene Name」のつづりがシーン名と一致していない。
→ 大文字小文字まで完全一致させよう（例: Zone1）。

原因3: 「Transition Destination」のタグが、出口側の「Destination Tag」と合っていない。
→ 両方を同じアルファベットにそろえる。

💡 15分たっても直らないときは、次の「AIを味方につけよう」の方法で相談してみよう。

5-6. 🤖 AIを味方につけよう

今日の「遷移」は設定する項目が多いぶん、どれか1つ抜けているだけで動かないのが特徴。
「自分が何をどう設定したか」を整理して伝える練習に、ぴったりの題材だよ。

この章でのAIの使い所

✅ 良い使い方 — 自分で15分考えてわからなかったら聞く

  • ワープが動かない: 「Transition Type」「Transition When」「各スロットに何を入れたか」を書いて聞くと、答えが正確になる
  • アイデア相談: 「ワープの面白い使い方」のようなレベルデザインの相談はAIの得意分野

そのままコピペで使えるプロンプト（AIへの指示文）の例:

Unity 2022.3の2D Game Kitで、同じシーン内のテレポートを作っている中学生です。
TransitionStartのTransitionPointコンポーネントに、Transitioning Game Object＝Ellen、Transition Type＝Same Scene、Destination Transform＝TransitionEnd、Transition When＝On Trigger Enter を設定しました。
でも入口に触れてもワープしません。確認すべきことを、順番に1つずつ教えてください。
2Dアクションゲームで「ワープ」を使った面白いステージのアイデアを、初心者が作れる範囲で3つ提案してください。
隠し部屋やショートカットなど、遊ぶ人がワクワクする使い方がいいです。

🚫 やめておこう

  • 設定を自分で見直さずに、AIの言う通り全部作り直す（まず自分の設定と答えを見比べるのが上達の近道）
  • AIの答えを動かして確かめずに信じる（必ず自分の画面で再生して確認）
