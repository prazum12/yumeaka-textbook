第2章

自分のステージを作ろう 〜タイルでペイント〜

📅 第2回の講習で使います（日付が決まったらここに記載）
💻 使っているPC:

🎯 この章でできるようになること

  • Kit Toolsで自分専用の新しいシーン（ステージ）を作れる
  • Tile Palette（タイルパレット）を開いて、使うタイルを選べる
  • タイルをペイントして、地面や足場を自由に描ける（消し方もわかる）
  • ▶で再生して、自分のステージをEllenで歩き回れる

前回はThe Explorerで「遊ぶ側」だったキミ。
今日からは「作る側」です。
まっさらな世界に、絵を描くみたいにステージをペイントしていくよ。

2-1. Kit Toolsで新しいシーンを作ろう

2D Game Kitには、ステージ作りをすぐ始められる専用メニュー「Kit Tools」が付いています。
これで作った新しいシーンには、Ellen（主人公）・小さな地面・体力UI・カメラなど、動き回ったり敵を攻撃したりするのに必要なものが最初からセットされているんだ。
「ゲームが動く最小セット付きの白紙」がもらえる、と思えばOK。

1Kit Tools > Create New Scene... を選ぶ

my-gamekit プロジェクトを開いて、上部メニューの「Kit Tools」をクリック →「Create New Scene...」を選びます。
（Kit Toolsメニューは、2D Game Kitをインポートすると現れる特別メニューだよ）

2シーンに名前を付けてCreate

小さなウィンドウが開くので、「New Scene Name」の欄に好きなシーン名を入力して「Create」をクリック。

⚠️ シーン名は MyStage1 のような半角英数字にしよう。
日本語やスペース入りの名前は、あとでエラーの原因になることがあります。

3できあがったシーンを確認する

新しいシーンがプロジェクト内に作られて、そのままエディタで開きます。
シーンビューを見てみよう。Ellenと小さな地面がもう置いてあるはず。
（このシーンは、ゲームに含めるシーンのリスト「Build Settings」にも自動で登録されます。今は「へえ」でOK）

2-2. Tile Paletteを開こう

ステージの地面や足場は、タイルという四角いブロックを並べて作ります。
Unityのタイルマップという機能で、方眼紙のマス目にスタンプを押していくイメージ。
そのスタンプ（タイル）を選ぶ道具箱が「Tile Palette（タイルパレット）」です。

1Window > Tile Palette を開く

上部メニューから「Window」→「Tile Palette」の順に選びます。
（メニューの場所がバージョンによって Window > 2D > Tile Palette の場合もあるよ）

こんなウィンドウが開けば成功。
シーンビューの横など、見やすい場所にドラッグして置いておこう。

2タイルセット「Tileset_Gamekit」を選ぶ

Tile Paletteの上のほうにあるタイルセットメニュー（ドロップダウン）をクリックして、「Tileset_Gamekit」を選びます。
すると、使えるタイルがずらっと表示されるよ。

タイルは大きく2種類。
気分やステージの雰囲気で使い分けよう。

タイルセット雰囲気
Grassy Rocks（草が生い茂る岩）緑ゆたかな自然のステージ。The Explorerの序盤の雰囲気
Alien Structure（異惑星文明の建造物）なぞの遺跡・SFっぽいステージ。終盤のダンジョン向き
スクリプタブルタイル … このキットのタイルはちょっとかしこくて、置いた場所に合わせて絵が自動で変わる仕組み（地面のはし、角、内側などを勝手に判断）。
だから適当に塗っても、それっぽい地形になるんだ。

2-3. ステージをペイントしよう

準備完了！
ここからは、お絵かきの時間です。

1タイルを選んで、シーンに塗る
  • Tile Paletteの中で、使いたいタイルをクリックして選ぶ
  • Tile Palette上部の道具から筆（Paint Brush）が選ばれていることを確認
  • シーンビューの上で左クリック＋ドラッグすると、タイルがペイントされる！
2まちがえたら Shift＋クリックで消す

塗りすぎた？ 大丈夫。
Shiftキーを押しながら左クリック（またはドラッグ）すると、タイルを消せます。
「Shift＝消しゴム」と覚えよう。

3ステージの形を作る

Ellenがスタートする地面から、右に向かってコースを作ってみよう。

  • 歩く地面を横にのばす
  • ジャンプで乗る足場を、少し高い位置に浮かせて置く
  • 落ちたらこわい谷や、のぼっていく階段も作れる

💡 できたら忘れずに保存（Ctrl + S⌘ + S）！

2-4. 再生してテストしよう

ステージができたら、Ellenを歩かせてみよう。
作っては遊び、遊んでは直す。
このくり返しがゲーム作りの心臓部です。

1▶（再生ボタン）を押す

操作は前回と同じ。
A/D で移動、スペース でジャンプ、O/K で攻撃、F12 で武器切り替えだよ。

2開発者の目でチェックする
  • ジャンプで届かない足場はない？（届かない＝そこから先に進めない）
  • 歩いていて、ひっかかる場所はない？
  • 落ちたら戻れない穴はない？

気になったところは、▶を停止してからタイルを塗り直そう。
再生中に直しても、停止すると元に戻っちゃうんだったね。

2-5. 💪 やってみよう

課題1: ジャンプで渡れる自分のステージを1画面分作ろう

スタート地点からゴール（自分で決めた右はしの地点）まで、ジャンプを3回以上使わないとたどり着けないステージを1画面分作ろう。
作ったら必ず自分でテストプレイして、最後まで行けることを確認してね。

ヒントを見る

Ellenのジャンプ力は「横に2〜3マス、上に2マスくらい」が目安。
まず1つ足場を置いて再生→届くか確認→次を置く、と少しずつ作ると失敗しないよ。

課題2: 2種類のタイルで雰囲気チェンジ

ステージの前半を Grassy Rocks、後半を Alien Structure で塗って、「自然エリアから遺跡エリアに入っていく」ような景色の変化を作ってみよう。

ヒントを見る

境目は、段差や谷にすると「エリアが変わった感」が出るよ。
タイルはTile Paletteでクリックして選び直すだけで切り替えられる。

課題3（チャレンジ）: わざと「ちょうどいい難しさ」を作ろう

「ギリギリ届くジャンプ」を1か所だけ入れて、となりの人（や講師）にテストプレイしてもらおう。
「難しいけどクリアできた！」と言ってもらえたら大成功。
むずかしすぎたら、足場を半マスずつ近づけて調整しよう。

ヒントを見る

プロのゲームデザイナーも、こうやって他の人に遊んでもらって難しさを調整しています。
「自分はクリアできるけど、初見の人は？」という目線が大事。

2-6. 🚨 つまずきポイント集

メニューに「Kit Tools」がない

原因: 2D Game Kitがまだインポートされていないプロジェクトを開いている。
または、インポートが途中で失敗している。

直し方: 開いているのが my-gamekit か確認。
合っているのに出ないときは、第1章の手順でPackage Managerからもう一度Importしよう。

シーンを作ろうとしたらエラーが出た（名前が原因かも）

原因: シーン名に日本語・スペース・記号が入っている。

直し方: MyStage1 のような半角英数字だけの名前で作り直そう。

クリックしてもタイルが塗れない

原因1: Tile Paletteでタイルを選んでいない。
→ 塗りたいタイルをクリックして選んでから塗る。

原因2: 道具が筆（Paint Brush）になっていない。
→ Tile Palette上部の筆アイコンをクリック。
キーボードの B でも筆に切り替えられるよ。

原因3: タイルセットメニューが「Tileset_Gamekit」になっていない。
→ 2-2の手順で選び直す。

タイルを消したいのに消せない

直し方: Shiftを押しながら左クリック（またはドラッグ）。
Shiftを先に押してからクリックするのがコツ。
広い範囲をまとめて消したいときは、Shiftを押したままドラッグしよう。

ペイントは終わったのに、クリックするたびタイルが塗られちゃう

原因: まだペイントモード（筆）のまま。

直し方: Tile Paletteを閉じるか、エディタ左上の移動ツール（キーボードの W）に切り替えよう。
「塗り終わったらW」と覚えておくと安全。

再生したらEllenが落ちて消えていく

原因: Ellenのスタート位置の下に地面がない。
（最初からある小さな地面を消してしまった場合もこうなるよ）

直し方: ▶を停止して、Ellenの真下にタイルを塗ろう。
Ellenがどこにいるかわからなくなったら、ヒエラルキーで Ellen をダブルクリックすると画面がズームするよ。

ジャンプしても足場に届かない

原因: 足場が高すぎ・遠すぎ。

直し方: Shift＋クリックで足場を消して、1〜2マス低い（近い）位置に塗り直す。
「上に2マス・横に2〜3マス」を目安に調整しよう。

💡 15分たっても直らないときは、次の「AIを味方につけよう」の方法で相談してみよう。

2-7. 🤖 AIを味方につけよう

今日から「作る側」の質問ができるようになったね。
AIに聞くときは、「何を使って」「何をしようとして」「どうなったか」の3点セットで伝えると、答えの正確さがぐっと上がるよ。

この章でのAIの使い所

✅ 良い使い方 — 自分で15分考えてわからなかったら聞く

  • 操作のトラブル: タイルが塗れない・消せないなどの症状を具体的に伝える
  • レベルデザインの考え方: 「面白いステージって何が違うの？」のような設計の質問はAIの得意分野

そのままコピペで使えるプロンプト（AIへの指示文）の例:

Unity 2022.3の2D Game Kitでステージを作っている中学生です。
Tile Paletteでタイルを選んでいるのに、シーンビューをクリックしてもタイルが塗れません。
確認すべきことを、順番に1つずつ教えてください。
2Dアクションゲームのステージを初めて作っています。
「最初はやさしく、だんだん難しく」なるステージにしたいです。
1画面分のステージでできる工夫を、初心者向けに3つ提案してください。

🚫 やめておこう

  • ステージの形を丸ごとAIに決めてもらう（キミのステージはキミのアイデアで！ 相談はOK、丸投げはNG）
  • AIの答えを動かして確かめずに信じる（必ず自分の画面でテストプレイして確認）
